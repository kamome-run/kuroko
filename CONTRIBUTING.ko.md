# 기여 가이드라인

[![Japanese](https://img.shields.io/badge/lang-日本語-white.svg)](CONTRIBUTING.md) [![English](https://img.shields.io/badge/lang-English-blue.svg)](CONTRIBUTING.en.md) [![Chinese](https://img.shields.io/badge/lang-中文-red.svg)](CONTRIBUTING.zh.md) [![Traditional Chinese](https://img.shields.io/badge/lang-繁體中文_🇹🇼-blue.svg)](CONTRIBUTING.zh-TW.md)

Kuroko 프로젝트에 기여해 주셔서 감사합니다!
버그 보고, 기능 제안, 풀 리퀘스트(Pull Request) 등 모든 형태의 기여를 환영합니다.

Issue 또는 Pull Request를 작성할 때 모국어를 사용하셔도 좋지만, 반드시 영어 번역을 함께 포함해 주십시오 (기계 번역도 가능합니다).

## 버그 보고

버그를 발견하면 GitHub Issues에 보고해 주세요.
보고할 때 다음 정보를 포함해 주시면 도움이 됩니다.

- 사용 중인 OS 및 버전 (예: ChromeOS 버전)
- Kuroko 버전
- 재현 단계
- 예상되는 동작과 실제 동작
- 오류 로그 또는 스크린샷

## 기능 제안

새로운 기능에 대한 아이디어가 있으면 GitHub Issues에 제안해 주세요.
어떤 기능이며 왜 필요한지 자세히 설명해 주세요.

## 풀 리퀘스트 (Pull Request)

코드 변경을 제안하려면 다음 단계를 따르세요.

1. 저장소를 포크(Fork)합니다.
2. 새 브랜치를 만듭니다 (`git checkout -b feature/your-feature-name`).
3. 변경 사항을 커밋합니다 (`git commit -m 'Add some feature'`).
4. 브랜치를 푸시합니다 (`git push origin feature/your-feature-name`).
5. 풀 리퀘스트(Pull Request)를 생성합니다.

## 라이선스

이 프로젝트에 기여함으로써 귀하는 귀하의 기여가 [LICENSE](LICENSE) 파일에 명시된 라이선스에 따라 배포된다는 데 동의하는 것으로 간주됩니다.
