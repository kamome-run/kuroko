# 貢獻指南

[![Japanese](https://img.shields.io/badge/lang-日本語-white.svg)](CONTRIBUTING.md) [![English](https://img.shields.io/badge/lang-English-blue.svg)](CONTRIBUTING.en.md) [![Chinese](https://img.shields.io/badge/lang-中文-red.svg)](CONTRIBUTING.zh.md) [![Korean](https://img.shields.io/badge/lang-한국어-green.svg)](CONTRIBUTING.ko.md)

感謝您有興趣為 Kuroko 專案做出貢獻！
我們歡迎各種形式的貢獻，包括錯誤報告、功能建議和拉取請求 (Pull Requests)。

在提交 Issue 或 Pull Request 時，您可以使用您的母語，但請務必同時附上英文翻譯（可以使用機器翻譯）。

## 錯誤報告

如果您發現錯誤，請在 GitHub Issues 上報告。
報告時，請包含以下資訊：

- 您使用的作業系統和版本（例如 ChromeOS 版本）
- Kuroko 版本
- 重現步驟
- 預期行為和實際行為
- 錯誤日誌或螢幕截圖

## 功能建議

如果您有新功能的想法，請在 GitHub Issues 上提出建議。
請詳細說明該功能是什麼以及為什麼需要它。

## 拉取請求 (Pull Requests)

如果您想提議程式碼更改，請按照以下步驟操作：

1. Fork 本儲存庫。
2. 建立一個新的分支 (`git checkout -b feature/your-feature-name`)。
3. 提交您的更改 (`git commit -m 'Add some feature'`)。
4. 推送分支 (`git push origin feature/your-feature-name`)。
5. 建立一個拉取請求 (Pull Request)。

## 授權

透過為本專案做出貢獻，即表示您同意您的貢獻將在 [LICENSE](LICENSE) 文件中指定的授權下發布。
