Kuroko プロジェクトへの貢献に興味を持っていただきありがとうございます！
バグ報告、機能追加の提案、プルリクエストなど、あらゆる形の貢献を歓迎します。

IssueやPull Requestを投稿する際は、日本語で記述してください。
機械翻訳で構いませんので、併記で英語でも記述していただけると一層ありがたいです。

## バグ報告

バグを発見した場合は、GitHub Issues で報告してください。
報告の際は、以下の情報を含めていただけると助かります。

- 使用している OS とバージョン (ChromeOS のバージョンなど)
- Kuroko のバージョン
- 再現手順
- 期待される動作と実際の動作
- エラーログやスクリーンショット

## 機能追加の提案

新しい機能のアイデアがある場合は、GitHub Issues で提案してください。
どのような機能で、なぜそれが必要なのかを詳しく説明してください。

## プルリクエスト

コードの変更を提案する場合は、以下の手順に従ってください。

1. リポジトリをフォークします。
2. 新しいブランチを作成します (`git checkout -b feature/your-feature-name`)。
3. 変更をコミットします (`git commit -m 'Add some feature'`)。
4. ブランチをプッシュします (`git push origin feature/your-feature-name`)。
5. プルリクエストを作成します。
